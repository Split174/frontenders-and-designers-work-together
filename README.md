# frontenders-and-designers-work-together

Шаблон репозитория для фронтово-дизайнерского хакатона в Челябинске.

![Дизайн-завтрак №39](cover.jpg)

User Story Map

### User Story Map (USM): Поиск Ж/Д билетов

**Персона:** Путешественник (планирует отпуск с жесткими рамками ~14 дней).
**Цель:** Найти билеты, укладывающиеся в график отпуска, максимально быстро.

#### Backbone (Хребет карты — Активности пользователя)
`Инициализация поиска` -> **`Выбор дат поездки`** -> `Просмотр выдачи`

*(В рамках задачи мы фокусируемся на активности **Выбор дат поездки**)*

#### Срез релиза (MVP - Реализовано в компоненте TicketSearch)

| Пользовательская задача (User Task) | Пользовательские истории (User Stories) | Реализация в коде |
| :--- | :--- | :--- |
| **1. Просмотр доступности** | Как пользователь, я хочу видеть календарь на 2 месяца вперед, чтобы планировать отпуск на стыке месяцев. | Рендер двух месяцев (`renderMonth(0)`, `renderMonth(1)`). |
| | Как пользователь, я не хочу видеть недоступные (прошедшие) даты. | Блокировка дат `date < today`, стиль `opacity: 0.3`. |
| **2. Быстрый выбор (Smart Search)** | **Как отпускник, я хочу в один клик выбрать период "2 недели", чтобы не считать дни вручную.** | Панель пресетов: `handlePresetClick(14)`. |
| **3. Ручной выбор дат** | Как пользователь, я хочу выбрать дату "Туда" и "Обратно" кликами по сетке. | Логика `handleDateClick` с установкой `startDate` и `endDate`. |
| | Как пользователь, я хочу наглядно видеть весь диапазон моего отпуска. | Визуализация полосы (`rangeStart`, `inRange`, `rangeEnd`). |
| **4. Гибкость маршрута** | Как пользователь, я хочу иметь возможность купить билет только в одну сторону (например, обратно на самолете). | Возможность выбрать только `startDate`. Кнопка "Очистить" сбрасывает выбор. |
| **5. Подтверждение** | Как пользователь, я хочу видеть выбранные даты в текстовом поле перед поиском, чтобы убедиться в правильности. | `formatDateDisplay` внутри инпута (например: "10 дек — 24 дек"). |
